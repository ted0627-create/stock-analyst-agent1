# 종목 분석 AI 에이전트 (Stock Analyst Agent)

증권사 리서치센터를 모사한 멀티 에이전트 시스템. Anthropic의 **orchestrator–workers** 패턴(《Building Effective Agents》)을 따른다. 총괄 오케스트레이터가 작업을 분해해 전문 워커 서브에이전트에 위임하고, 결과를 하나의 리서치 리포트로 종합한다.

## 구조

```
사용자 요청 ("삼성전자 분석해줘" / "반도체 추천주")
        │
        ▼
┌──────────────────────────────┐
│ stock-analyst-orchestrator   │  ← 계획·위임·종합 (모델: opus)
│ (Head of Research)           │
└──────────────────────────────┘
        │  병렬 위임 (fan-out)
   ┌────┬────┬────┬────┬─────────────┐
   ▼    ▼    ▼    ▼    ▼
 펀더   기술  시장  리스크  추천전략
 멘털   적    심리         (스크리닝/
                          최종의견)
```

| 에이전트 | 역할 | 모델 |
|----------|------|------|
| `stock-analyst-orchestrator` | 요청 분류, 워커 조율, 리포트 종합, 면책 부착 | opus |
| `fundamental-analyst` | 밸류에이션·재무·성장성 | sonnet |
| `technical-analyst` | 가격/거래량 추세·모멘텀·지지저항 | sonnet |
| `market-sentiment-analyst` | 산업 동향·뉴스/공시·시장 심리 | sonnet |
| `risk-analyst` | 변동성·유동성·재무/규제 리스크 (악마의 변호인) | sonnet |
| `recommendation-strategist` | 종목 스크리닝 + 최종 투자의견 도출 | sonnet |

## 데이터 소스

KRX 공식 데이터를 `korean-stock-search` 스킬(k-skill-proxy)로 조회한다. 사용자는 `KRX_API_KEY` 없이 사용 가능. 재무·뉴스·산업 정보는 워커가 WebSearch/WebFetch로 보강한다.

## 설치 / 위치

서브에이전트 파일은 `.claude/agents/*.md` 에 있어 Claude Code가 자동 인식한다. 다른 프로젝트에서 쓰려면 같은 6개 파일을 해당 프로젝트의 `.claude/agents/` 또는 `~/.claude/agents/` 로 복사한다.

## 사용 예

- "삼성전자 종합 분석하고 매수 의견 줘" → 단일 종목 파이프라인
- "SK하이닉스랑 삼성전자 비교해줘" → 병렬 분석 + 비교표
- "배당 잘 주는 코스피 종목 추천해줘" → 스크리닝 → 상위 후보 심층 분석
- "005930 오늘 시세만 알려줘" → 위임 없이 스킬 직접 조회

## 설계 원칙

- **사실과 추정 분리**: 모든 수치에 기준일·출처. KRX 정량값과 웹 보강값을 구분.
- **양면 논거**: 강세/약세 양쪽을 모두 제시해 확증편향 회피. 리스크는 항상 동반.
- **의도 가중치**: 장기/단기/배당 등 사용자 의도에 따라 종합 가중치를 달리함.
- **면책**: 모든 리포트는 정보 제공 목적이며 투자 권유·자문이 아님. 오케스트레이터가 부착.
